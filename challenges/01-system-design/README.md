# Challenge 01 — System Design & Implementation

## Time: 60 minutes

## Scenario

You have been given a product spec for a **Bookmark/Link Management API**. Your task
is to design and build the entire service from scratch, starting from the provided
starter directory.

## Instructions

1. Read `spec.md` carefully — it contains all functional and technical requirements
2. Start from `starter/` — it has a pre-configured `package.json` and `.env.example`
3. Design the architecture, implement the API, and write tests
4. The API should be fully functional and all tests should pass when you are done

## Getting Started

```bash
cd starter/
cp .env.example .env
npm install
# Start building!
```

## What We Evaluate

- **Problem decomposition**: How do you break down the spec into implementable pieces?
- **Architecture decisions**: How do you structure the code? What patterns do you choose?
- **AI delegation**: What parts do you hand to AI? What do you decide yourself?
- **Incremental development**: Do you test as you go, or only at the end?
- **Completeness**: Does the final API meet all requirements in the spec?

## Deliverable

A working Express.js API with:
- All endpoints from the spec implemented
- SQLite database with proper schema
- Input validation on all endpoints
- Error handling middleware
- At least 15 integration tests that pass
- Clean, well-organized code structure
